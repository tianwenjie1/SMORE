## 第一优先级：必做（让主表和敏感性分析一致）

### 1.1 用 rate=0.2 重跑主表

**为什么必做**：rate 搜索证明 0.2 比 0.1 好，但当前主表用的是 0.1。论文里主表数字必须和敏感性分析结论一致，否则审稿人会质疑。用 0.2 重跑后，主表数字能再涨 ~0.3%，且逻辑自洽。

**改动**：把 `run_keyexperiments.sh` 里 MDR/FBG+MDR 的 `modality_dropout_rate=0.1` 改成 `0.2`，重跑 36 个。

**预计**：~7 小时（2 GPU 并行）

**命令**：
```bash
# 服务器上
sed -i 's/modality_dropout_rate=0.1/modality_dropout_rate=0.2/g' run_keyexperiments.sh
nohup ./run_keyexperiments.sh > run_key02.log 2>&1 &
```

## 第三优先级：可选（看投稿要求）

### 3.1 外部基线对比

**为什么可选**：当前 baseline 只是原始 SMORE。要证明方法有竞争力，需和其他多模态推荐模型比（BM3、SLMRec、LATTICE、MMGCN 等）。

**可行性**：SMORE 基于 MMRec 框架，这些模型可能有现成实现。但每个模型要单独跑通调参，工作量大（2~3 天）。

**判断**：
- 如果投顶会（WSDM/SIGIR/CIKM）→ 必须做
- 如果是小论文/课程论文/国内期刊 → 可不做，baseline 用原始 SMORE 即可

### 3.2 更多 seed

当前 3 seed。要更强统计显著性可加到 5 seed（加 2023、2025）。但 3 seed 已够用，边际收益低。

### 3.3 drop 强度敏感性

当前 drop_image/drop_text 是完全置零。可改成部分屏蔽（保留 X% 特征）。但 drop 场景所有方法都不掉点，价值不高，不建议。

## 不建议做

- **加第五个创新点**：够了，再加会稀释主线
